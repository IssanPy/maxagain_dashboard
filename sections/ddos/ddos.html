🧠 TCP/IP Stack Internals – The Exhaustion Point

TCP State Machine & SYN Queue

The TCP three‑way handshake allocates memory in the SYN backlog. When an attacker sends a flood of SYN packets without completing the handshake (half‑open connections), the backlog fills, and legitimate clients can no longer connect. This is the classic SYN flood.

# Check SYN backlog size
sysctl net.ipv4.tcp_max_syn_backlog

Retransmissions & Congestion Collapse

TCP retransmits lost packets, doubling the backoff timer each time (exponential backoff). Attackers can trigger excessive retransmissions by dropping selective ACKs or manipulating RTT. Congestion collapse happens when the network spends all bandwidth on retransmissions.

# Simulate packet loss with netem
tc qdisc add dev eth0 root netem loss 10%

Conntrack Table & Socket Exhaustion

The kernel's connection tracking table (conntrack) has a finite size. Once full, new connections are dropped. Attackers can exhaust it by spoofing many unique source IPs. TIME_WAIT states also consume memory.

# Monitor conntrack usage
conntrack -C
sysctl net.netfilter.nf_conntrack_max

SYN Cookies & Mitigations

SYN cookies encode the initial sequence number cryptographically, avoiding the need to store half‑open connections. This is the kernel's built‑in defense against SYN floods. Enable it with sysctl net.ipv4.tcp_syncookies=1.

⚡ HTTP/2 & QUIC/HTTP3 – The Modern Flood

HTTP/2 Rapid Reset Attack (CVE‑2023‑44487)

An attacker opens a single HTTP/2 connection, creates thousands of streams, and immediately sends RST_STREAM for each. The server processes every stream request before the reset, exhausting CPU. Cloudflare, AWS, and Google were hit by attacks exceeding 398 million requests per second using this technique.

# Conceptual: rapid stream creation and cancellation
for i in range(0, 1000):
    send_headers(stream_id=i)
    send_rst_stream(stream_id=i)

QUIC Connection Migration & Handshake Abuse

QUIC uses connection IDs, allowing clients to change IP addresses without renegotiation. Attackers can spoof migration packets, causing the server to spend resources verifying them. Initial handshake packets can also be amplified.

# QUIC amplification test
hping3 -2 -p 443 -a  --flood

HPACK/QPACK Compression Abuse

HTTP/2 and HTTP/3 use HPACK and QPACK for header compression. Attackers can send many headers with high entropy, causing the compressor to allocate excessive memory or CPU.

🌍 Anycast, CDN & Edge Architecture

Anycast Routing & Traffic Diffusion

Anycast allows multiple servers around the world to share the same IP address. BGP routes traffic to the nearest (topologically) instance. This naturally diffuses volumetric attacks, as the flood is divided among many data centers.

Edge Shielding & Cache Bypass

CDNs protect origin servers with edge caches. Attackers use cache‑busting techniques (random query parameters, different URL paths) to force every request to the origin, bypassing the CDN entirely. This is a "cache miss storm."

# Cache bypass via random parameters
for i in range(0, 100000):
    send_get("/page?nocache=" + str(i))

BGP Propagation & Internet Topology

Anycast depends on BGP announcements. Attackers can potentially hijack the prefix with a more specific route (BGP hijack) to redirect traffic to their own infrastructure. Understanding internet topology is key to understanding Anycast resilience.

🗺️ BGP & Internet Routing Warfare

Route Hijacking & Leaks

Malicious ASes can announce more specific prefixes, stealing traffic meant for the victim. Route leaks can cause traffic to take longer, congested paths. Both have been used as DDoS amplifiers.

RTBH & Flowspec for Mitigation

Remotely Triggered Black Hole (RTBH) uses BGP to drop all traffic to a target at the network edge. Flowspec allows more granular filtering based on packet characteristics. These are the backbone of carrier‑grade DDoS mitigation.

# RTBH configuration (conceptual)
router bgp 65001
network 192.0.2.1/32 route-map blackhole

Internet Exchange Points & Transit

IXPs provide peering; transit providers carry global traffic. DDoS can saturate peering links, forcing traffic through expensive transit. Understanding this helps target attacks where they hurt most.

🧠 Kernel Network Stacks – eBPF & XDP

epoll, Socket Exhaustion & Conntrack

Servers use epoll for high‑performance I/O, but file descriptor limits and socket buffers can be exhausted. Conntrack table overflow is a classic DDoS vector. Modern kernels use eBPF to offload filtering to the earliest possible point.

XDP – Express Data Path

XDP allows packet filtering directly in the NIC driver, before the kernel allocates any memory. It can drop malicious traffic at line rate (millions of packets per second per core). This is how hyperscale defenders survive Tbps attacks.

# Simple XDP program (C)
SEC("xdp")
int xdp_drop(struct xdp_md *ctx) {
    return XDP_DROP;
}

DPDK & NIC Offloading

Data Plane Development Kit (DPDK) bypasses the kernel entirely, processing packets in userspace at 100+ Gbps. Combined with XDP, it forms the backbone of modern mitigation appliances.

🤖 Botnet Operations Masterclass

Loader Infrastructure & Propagation

Botnets use loaders (cgi‑bin scripts, Telnet brute‑forcers) to infect new devices. Once enrolled, the bot beacons to a C2 server. Mirai's loader was a simple binary that brute‑forced 61 default credentials.

# Mirai credential list snippet
root:xc3511
root:vizxv
root:admin

Domain Generation Algorithms (DGA) & P2P Resilience

DGAs generate pseudo‑random domain names daily, making takedowns difficult. P2P botnets (e.g., Mozi) communicate directly between peers, eliminating central C2 entirely. Both require advanced tracking and sinkholing.

Monetization – Proxies, DDoS‑for‑Hire, Credential Stuffing

Botnets are businesses. Residential proxy services (e.g., Faceless, 911.re) rent out infected IoT devices. Booter services sell DDoS attacks by the minute. Stolen credentials are brute‑forced across banking sites. The economics drive the infections.

📈 Amplification Attack Science

Reflection & Amplification Mechanics

Attacker spoofs victim's IP, sends a small request to a vulnerable server (DNS, NTP, Memcached). The server sends a much larger response to the victim, multiplying the traffic.

# DNS amplification: 64‑byte query → 3000‑byte response
dig ANY isc.org @open_resolver -b

Recursive Amplification Chains

Attackers chain multiple reflectors together, creating exponential traffic growth. For example, a single packet to a CHARGEN server can trigger an infinite loop between two misconfigured hosts.

Modern Amplification Vectors

  • CLDAP – up to 70× amplification
  • WS‑Discovery – up to 150×
  • SSDP – up to 30×
  • QUIC reflection – emerging

🕸️ Layer 7 Resource Exhaustion – Killing Logic

Thread Pool & DB Connection Exhaustion

Attackers send slow HTTP POST requests (R.U.D.Y.), keeping all server threads busy. Alternatively, they flood expensive database queries (e.g., a recursive search) until the DB collapses.

# Slow POST attack (R.U.D.Y.)
while (true) send_byte_at_random_interval();

Expensive Regex & GraphQL Query Bombs

Regular expressions with catastrophic backtracking (ReDoS) can kill a server with a single request. GraphQL queries with deep nesting can cause exponential processing.

# Deeply nested GraphQL query
query {
  user(id:1) { friends { friends { friends { friends { id } } } } }
}

WebSocket & Realtime App Exhaustion

Opening thousands of WebSocket connections and sending fragmented frames keeps servers busy. Idle connection starvation eventually kicks legitimate users off.

🤖 AI‑Driven DDoS Warfare

Adaptive Attack Tuning & ML Evasion

AI bots analyze the target's defenses in real time, adjusting attack patterns to evade mitigation. Traffic is shaped to mimic legitimate user behavior, defeating rate limiting and behavioral detection. Reinforcement learning agents optimise attack vectors for maximum damage.

Synthetic Traffic Generation

Generative AI creates realistic HTTP headers, user agents, and mouse movements, making it nearly impossible to distinguish bots from humans. TLS fingerprint rotation further complicates detection.

🏢 ISP & Carrier‑Grade Defense

Scrubbing Centers & Arbor Networks

ISPs operate massive scrubbing centers that filter attack traffic using DPI and behavioral analysis. Solutions like Arbor Networks Peakflow redirect traffic via BGP and GRE tunnels, clean it, and reinject it.

# GRE tunnel to scrubbing center
ip tunnel add scrub0 mode gre remote  local

Carrier Filtering & Blackholing

Upstream providers can blackhole entire /32 prefixes during an attack. This sacrifices the target but protects the rest of the network. More advanced filtering uses Flowspec to drop only specific protocols or packet sizes.

📶 Telecom & 5G DDoS

GTP Flooding & PFCP Exhaustion

GTP (GPRS Tunneling Protocol) carries user traffic in mobile cores. An attacker on the S5/S8 interface can flood the SGW/PGW with GTP echo requests or modify session keys, crashing the data plane. PFCP (Packet Forwarding Control Protocol) can be abused to redirect traffic or exhaust UPF resources.

SCTP Attacks & Signaling Storms

SCTP (Stream Control Transmission Protocol) is used for signaling between 5G core functions. An unauthenticated attacker can send malformed SCTP packets to crash the AMF or SMF, disabling entire network slices.

RAN Congestion & UPF Overload

Massive device registrations can overload the RAN (Radio Access Network) or UPF (User Plane Function). 5G's network slicing can be exploited: a compromised eMBB slice can exhaust shared UPF resources, impacting uRLLC slices.

☁️ Cloud DDoS Engineering

AWS Shield Advanced & Google Cloud Armor

AWS Shield provides layer 3/4 protection and DDoS cost protection. Google Cloud Armor integrates with Cloud Load Balancing for adaptive protection. Both can absorb massive volumetric attacks.

# AWS Shield Advanced metrics
aws cloudwatch get-metric-statistics --namespace AWS/DDoSProtection --metric-name DDoSAttackBitsPerSecond

Autoscaling Collapse & Serverless Exhaustion

Attackers can trigger rapid autoscaling, costing millions in cloud resources. Serverless functions (AWS Lambda) can be invoked recursively until the account is throttled. Cloud DDoS becomes a financial attack.

Kubernetes API Exhaustion & Ingress Overload

Overloading the kube‑apiserver or ingress controller can take down an entire cluster. Use rate‑limiting annotations and dedicated ingress nodes.

💰 DDoS Economics – Why Attacks Happen

Booter/Stresser Ecosystems

Booter services sell DDoS attacks for as little as $5. They operate on top of compromised botnets or rented cloud resources. Many are used by gamers to kick opponents offline, but the infrastructure is the same used for extortion.

Ransom DDoS & Extortion

Attackers send a threatening email: "Pay 10 BTC or we'll take your site offline." They demonstrate a small attack, then demand payment. Most are bluffs, but some are backed by real capability.

Residential Proxy Monetization

Infected IoT devices are rented as residential proxies, allowing attackers to bypass geo‑restrictions and avoid IP blacklisting. This is a multi‑billion dollar illicit industry.

🔬 Packet Capture & Attack Forensics

PCAP Analysis & Flow Reconstruction

Use tcpdump to capture attack traffic, then analyze with Wireshark or Zeek. Reconstruct flows to identify spoofed IPs, amplification protocols, and attack signatures. NetFlow/IPFIX provides aggregated flow data without capturing every packet.

tcpdump -i eth0 -w attack.pcap
zeek -r attack.pcap

Attack Fingerprinting & Entropy Analysis

Different bot families have unique packet patterns (TTL values, window sizes, TCP options). Use entropy analysis to detect randomized fields. Tools like FastNetMon provide real‑time detection.

🕵️ Evasion Techniques – Hiding in Plain Sight

Low‑and‑Slow Attacks & Randomized Headers

Instead of high‑volume floods, send requests at a rate just above the detection threshold, blending with normal traffic. Randomize User‑Agent, Accept headers, and use residential proxies to appear as legitimate users.

TLS Fingerprint Rotation & CAPTCHA Bypass

Modern DDoS tools emulate browser TLS fingerprints (JA3/JA4) to avoid being flagged as bots. CAPTCHA farms and AI‑powered solvers defeat challenge‑response systems at scale.

📡 DNS Infrastructure Attacks

Authoritative DNS Exhaustion & NXDOMAIN Floods

Flooding authoritative DNS servers with queries for nonexistent domains (random subdomains) forces them to process NXDOMAIN responses, exhausting resources. This is how the 2016 Dyn attack took down major sites.

DNSSEC Overhead & Recursive Resolver Abuse

DNSSEC validation adds cryptographic overhead. Attackers can flood resolvers with signed queries for zones with complex chains, causing CPU exhaustion. Recursive resolvers can also be used for amplification.

📖 Internet‑Scale DDoS Incidents Dissected

Dyn DNS Attack (2016) – 1.2 Tbps Mirai

A Mirai botnet of ~100,000 IoT devices flooded Dyn's managed DNS infrastructure, taking down Twitter, Netflix, Reddit, and GitHub across the US East Coast. The attack used DNS water torture – random subdomain queries to bypass caching.

GitHub Memcached Amplification (2018) – 1.35 Tbps

Attackers exploited misconfigured Memcached servers with a 51,000× amplification factor. GitHub's DDoS protection (Akamai Prolexic) quickly rerouted traffic to scrubbing centers, mitigating the attack in under 10 minutes.

Cloudflare HTTP/2 Rapid Reset (2023) – 398M rps

The largest ever recorded application‑layer attack. Exploited HTTP/2 stream cancellation to overwhelm Cloudflare's edge servers. Cloudflare mitigated using targeted configuration changes and upstream collaboration.

Google DDoS Defense (2020) – 2.54 Tbps

A state‑sponsored actor launched a multi‑vector attack combining HTTP floods, DNS floods, and TCP SYN floods. Google's global load‑balancing and Maglev infrastructure absorbed it without service interruption.

Lessons from Every Incident

Anycast, real‑time monitoring, pre‑planned BGP blackholing, and kernel‑level filtering (XDP) are essential. Never rely on a single layer of defense.

🧠 The Network War Researcher's Mindset

“What resource is being exhausted?”

Always identify the bottleneck: SYN queue, conntrack table, thread pool, DB connections, upstream bandwidth. The entire attack vector is chosen to saturate that specific resource.

“What happens under partial exhaustion?”

Systems degrade gracefully or collapse catastrophically. Test your infrastructure by incrementally increasing load until failure. Understand the shape of the degradation curve.

“What assumptions does the protocol make?”

TCP assumes clients are honest. QUIC assumes connection migration is rare. DNS assumes queries are legitimate. Break those assumptions and the protocol fails.

The Network Warrior's Golden Rule

Defense is asymmetric: you must be right every time; an attacker only needs to succeed once. Design for resilience, not perfection. Practice chaos engineering.

🛡️ Advanced Hyperscale Defense Engineering

XDP Filtering & DPDK Mitigation

Drop malicious packets at the NIC level with XDP programs. Use DPDK to process good traffic in userspace at line rate. These techniques allow 100+ Gbps mitigation on a single commodity server.

Adaptive Rate Limiting & ML Detection

Use algorithms like leaky bucket and token bucket. Machine learning models detect attacks in real time by analyzing entropy, packet inter‑arrival times, and header patterns. Combine with behavioral fingerprinting.

Autonomous Mitigation Pipelines

Integrate attack detection (NetFlow, Zeek) with automated BGP Flowspec announcements to instantly drop attack traffic. Close the loop between detection and mitigation in under 10 seconds.

Challenge‑Response & Proof‑of‑Work

Use cryptographic puzzles (like Cloudflare's "I Am Under Attack" mode) to force clients to solve a problem before connecting. This slows down botnets but also annoys legitimate users.

Lab Safety Rule
