🧠 XNU – The Core of Apple’s Operating System

Mach Microkernel & BSD Layer

XNU (X is Not Unix) combines a Mach microkernel with a BSD subsystem. Mach handles tasks, threads, ports, and IPC. BSD provides the POSIX layer: file descriptors, sockets, and process management. IOKit is the device driver framework.

Attackers exploit Mach ports to transfer memory between processes, or use IOKit for kernel memory r/w. The task_for_pid() API is a prime target — it grants full task control if the attacker can bypass entitlement checks.

// Get task port for another process (requires entitlement)
task_for_pid(mach_task_self(), pid, &task_port);

Memory Zones, Launchd, and Syscalls

XNU's zone allocator manages kernel memory pools. Zone corruption can lead to kernel privilege escalation. Launchd (PID 1) spawns all processes; exploiting it can hijack the entire boot chain. Syscalls are the gateway from userland to kernel — many exploits start by finding a broken system call.

# List available syscalls
syscallbypid.pid 1

💪 ARM64 & Pointer Authentication

ARM64 Architecture for iOS

All modern iPhones run on ARM64 (A7+). Registers: X0‑X30, SP, PC, LR. Calling conventions: first 8 arguments in X0‑X7, return value in X0. Understanding stack frames and exception levels (EL0 user, EL1 kernel, EL3 Secure Monitor) is crucial for writing shellcode.

PAC (Pointer Authentication Codes)

Apple’s ARM64e extension adds cryptographic signing to pointers. The PACIASP instruction signs the return address and stack frame. Any tampering causes a fault. PAC is the main reason ROP chains are no longer trivial on iOS.

Bypasses: pointer spraying (guessing a valid PAC), signing gadgets (finding code that signs an attacker‑controlled pointer), or using a key leakage. Pangu team demonstrated a PAC bypass in 2019 by abusing an unauthenticated load.

; Signing the return address
PACIASP
; Verifying before return
AUTIASP

BTI & Kernel Mitigations

Branch Target Identification prevents arbitrary indirect jumps. KTRR (Kernel Text Readonly Region) makes kernel code immutable. SKEL (System Kernel Extension Library) virtualises kernel extensions. These are layers you must peel in a modern chain.

💥 Memory Corruption – The Heart of Exploitation

Use‑After‑Free, Type Confusion, OOB

Use‑after‑free (UAF) is the most common exploit primitive in WebKit and kernel. Type confusion (e.g., treating a JSCell as a different type) leads to arbitrary r/w. Out‑of‑bounds reads/writes in image parsers (ImageIO) or audio codecs can leak and corrupt memory.

// Pseudo‑exploit: dangling pointer after free
let obj = new Uint8Array(1024);
let dangling = obj; // reference kept
obj = null; // freed, but dangling still points to old memory

Race Conditions & Heap Grooming

Race conditions in XNU’s semaphore or file handling can be triggered by multiple threads. Heap grooming carefully allocates and frees objects to place a victim object next to a controlled overflow. This is the foundation of every modern browser exploit.

🌐 WebKit / Safari Exploitation

JavaScriptCore (JSC) & JIT Compilation

WebKit’s JavaScript engine has multiple tiers: LLInt (interpreter), Baseline JIT, DFG (Data Flow Graph) JIT, and FTL (Faster Than Light). Exploits often target the DFG JIT’s optimisations — a single incorrect type inference can lead to arbitrary code execution.

// Triggering JIT type confusion (simplified)
function jit_trigger(controlled) {
    let arr = [1.1, 2.2, 3.3]; // map as double array
    arr[controlled] = {}; // transition to object array
    return arr[0]; // type confusion if accessed as double
}

Sandbox Escape & BlastDoor Bypass

After RCE in the renderer, you’re still inside the WebContent sandbox. The next step is to escape to the kernel. This often involves exploiting another bug in a service reachable from the sandbox (e.g., IOSurface, Kernel Task). Apple’s BlastDoor service was introduced to filter iMessage content; bypassing it is a key step in zero‑click chains.

🔐 Secure Enclave Processor (SEP)

Architecture & Key Management

The SEP is a separate ARM processor with its own OS (SEPOS). It handles Touch ID, Face ID, passcode validation, and cryptographic keys. It stores the device UID key (burned at manufacture) and GID key (shared across models). No software running on the AP (Application Processor) can read these keys — only the SEP can access them.

SEP Exploitation History

In 2017, the pangu team demonstrated the first known SEP attack by exploiting a heap overflow in the SEP’s firmware. In 2020, checkm8 (bootrom exploit) was combined with a SEP firmware vulnerability to extract passcode‑protected data from iPhone X. These attacks require physical access and sophisticated hardware.

📦 Sandbox Profiles & Entitlements

Seatbelt (Sandboxd)

Every iOS process has a sandbox profile enforced by the kernel. It restricts file access, syscalls, and IPC. Profiles are written in SBPL (Sandbox Profile Language) and stored in /usr/share/sandbox/.

(allow file-read* (path "/private/var/mobile/Containers"))

Entitlements & TCC

Entitlements are XML‑like dictionaries embedded in app signatures. They grant capabilities like com.apple.security.exception.files.absolute-path.read. TCC (Transparency, Consent, and Control) manages privacy prompts (camera, contacts). Attackers abuse misplaced entitlements to escape the sandbox.

# Check entitlements of an app
codesign -d --entitlements - /path/to/app.app

🔓 Jailbreak Internals – Breaking the Chain

bootrom, iBoot, and Kernel Patches

A bootrom exploit (like checkm8) is unpatchable because it’s in hardware ROM. iBoot is the second‑stage bootloader; it loads the kernel, which verifies AMFI (Apple Mobile File Integrity). Jailbreaks patch the kernel to disable AMFI, load unsigned code, and remount the root filesystem as read‑write.

// Kernel patch: disable code signing (conceptual)
patch_kernel_amfi();         // allow unsigned code
patch_kernel_sandbox();      // relax sandbox
remount_rootfs_rw();         // access full filesystem

Trust Cache Injection & Substrate

The trust cache is a list of allowed code hashes. Jailbreaks inject their own hashes to load tweak libraries (Substrate, Substitute, libhooker). Semi‑tethered jailbreaks require a sideloaded app to re‑exploit after each reboot; untethered ones persist across boots.

⚡ Frida Masterclass – Instrumentation & Bypass

Objective‑C & Swift Hooking

Frida injects a JavaScript runtime into the target process. Hook Objective‑C methods with ObjC.classes, Swift methods with Module.findExportByName. Override implementations to bypass jailbreak detection, SSL pinning, or extract secrets.

// Frida script: bypass SSL pinning (generic)
Java.perform(() => { /* Android only; iOS uses ObjC */ });
// iOS ObjC hook
ObjC.classes.NSURLSession['- dataTaskWithRequest:completionHandler:'].implementation = function() {
    console.log("Intercepted request: " + arguments[0]);
    return this.dataTaskWithRequest_completionHandler_(arguments[0], arguments[1]);
};

Keychain Dumping & Anti‑Frida Detection

Apps can detect Frida by scanning memory for Frida’s agent strings or by checking for open ports. Bypass these checks by patching the detection functions with Frida itself. Dump the keychain using security dumper or Objection.

objection -g com.app explore
ios keychain dump

🔬 iOS App Reverse Engineering

IPA Structure, Info.plist, and Mach-O Binaries

An IPA is a ZIP containing a Payload folder with the .app bundle. Info.plist holds the app’s entitlements, supported devices, and URL schemes. The main binary is a Mach‑O file. Use class‑dump to extract Objective‑C headers.

class-dump -H /path/to/app.app/app -o headers/

Hopper, Ghidra, and Strings Analysis

Disassemble the binary with Hopper or Ghidra. Look for hardcoded URLs, API keys, and crypto constants. Use strings on the binary to find readable secrets.

strings ./app_binary | grep -i "http\|api\|secret\|token"

📄 Mach-O – The iOS Executable Format

Headers, Load Commands, and Sections

A Mach‑O file starts with a magic number (MH_MAGIC_64). The header defines the CPU type and file type. Load commands describe the layout: segments (__TEXT, __DATA), symbol table, dynamic linker info. The __LINKEDIT segment holds the code signature.

otool -l /bin/ls   # list load commands
jtool --sig /path/to/app   # check code signature

Encryption & Code Signatures

App Store apps are encrypted (FairPlay). They must be decrypted before analysis using tools like dumpdecrypted or bagbak. The code signature (LC_CODE_SIGNATURE) is verified by AMFI at runtime.

# Decrypt an app on a jailbroken device
dumpdecrypted /var/mobile/Containers/Bundle/Application/.../app.app

Pegasus is the most advanced iOS spyware. It uses zero‑click exploits (FORCEDENTRY) to inject code via iMessage. Once installed, it can read messages, listen to calls, activate the camera, and exfiltrate data. It uses rootless persistence to survive reboots without a jailbreak.

Operation Triangulation & Supply‑Chain Abuse

Kaspersky discovered an iOS zero‑click chain in 2023 that used an iMessage vulnerability to deploy spyware. Attackers also abuse enterprise certificates to distribute apps outside the App Store, bypassing Apple’s review process.

📡 iOS Network Security & Interception

ATS, SSL Pinning, and Proxy Detection

App Transport Security (ATS) forces HTTPS by default. Apps can implement SSL pinning to prevent MITM attacks. However, many still don’t. Use mitmproxy or Burp Suite with a self‑signed CA on a jailbroken device to intercept traffic. Frida can bypass pinning at runtime.

# mitmproxy setup for iOS
mitmproxy --ssl-insecure
# Set iOS proxy to :8080, install CA

HTTP/2, WebSockets, and Background Traffic

iOS apps often use long‑lived connections. Monitor background network activity with rvictl (iOS remote virtual interface) combined with Wireshark. This captures all traffic without a proxy.

rvictl -s 
# Now sniff on the created rvi interface with Wireshark

🛡️ Runtime Protections & How to Bypass

Jailbreak Detection Techniques

Apps check for jailbreak files (/Applications/Cydia.app), use sysctl to detect new process flags, or attempt to write to system directories. Frida hooks into these checks and forces them to return false.

// Frida: bypass a file‑based jailbreak check
var NSFileManager = ObjC.classes.NSFileManager.defaultManager();
Interceptor.attach(NSFileManager['- fileExistsAtPath:'].implementation, {
    onLeave: function(retval) {
        if (ObjC.Object(args[2]).toString().includes('Cydia')) {
            retval.replace(0);
        }
    }
});

Anti‑Debugging & Integrity Checks

ptrace(PT_DENY_ATTACH) prevents debuggers from attaching. Patch it or use debugserver on a jailbroken device. Integrity checks hash the app’s binary; bypass by hooking the hash function.

🔏 CoreTrust, AMFI & Code Signing

AMFI (Apple Mobile File Integrity)

AMFI enforces code signatures. It uses a kernel extension to verify every executable page load. Trust caches (static and dynamic) allow known hashes. CoreTrust is the daemon that manages trust evaluations for AppleID‑signed apps.

Enterprise Certificate Abuse & Sideloading

Attackers obtain enterprise certificates to sign malicious apps, bypassing the App Store. These certificates can be revoked by Apple, but not before thousands of devices are infected. AltStore and similar tools use this technique legitimately.

⛓️ Exploit Chains – The Full Attack Workflow

The Zero‑Click iMessage Chain

  1. WebKit RCE via a malicious PDF or image in iMessage.
  2. Sandbox Escape exploiting a kernel extension vulnerability.
  3. Kernel Exploit for arbitrary kernel r/w.
  4. PAC Bypass using signing gadgets.
  5. Persistence via a launch daemon or dylib injection.
  6. Payload Deployment – spyware, credential theft, surveillance.

Real chains like FORCEDENTRY (NSO Group) executed this entire sequence with no user interaction.

🔎 iOS Forensics & Data Extraction

Logical & Filesystem Acquisition

Logical acquisition uses iTunes backup (encrypted or unencrypted) to extract messages, photos, and app data. Filesystem acquisition requires a jailbreak or checkm8‑based exploit to access the full raw disk.

# Pull encrypted backup on macOS
idevicebackup2 backup /path/to/backup

Keychain, SQLite, and Crash Logs

The iOS keychain stores passwords, tokens, and certificates. Tools like Keychain Dumper extract them on jailbroken devices. SQLite databases (sms.db, call_history.db) contain communication records. Crash logs (.ips) can leak function addresses for ASLR bypass.

🧬 Swift & Objective‑C Runtime Reversing

Objective‑C Messaging & Selectors

ObjC uses a dynamic messaging system: [object method:argument] becomes objc_msgSend(object, selector, argument). Selectors are referenced by unique IDs. You can swizzle methods (replace implementations) at runtime, which is a powerful technique for hooking and analysis.

# Dump ObjC classes and methods
class-dump -H /path/to/app.app/app

Swift Metadata & Demangling

Swift uses name mangling: a method like AppDelegate.application(_:didFinishLaunchingWithOptions:) becomes _$s11AppDelegateC11application_29didFinishLaunchingWithOptionsSbSo13UIApplicationC_SDySo0mF0aGSgSgtF. Use swift demangle to restore readability.

xcrun swift-demangle _\$s11AppDelegateC11application_29didFinishLaunchingWithOptionsSbSo13UIApplicationC_SDySo0mF0aGSgSgtF

🔧 Hardware Security & Physical Attacks

SecureROM, Boot Chain, & DFU

SecureROM (bootrom) is the first code executed. It’s immutable — vulnerabilities here (like checkm8) are forever. DFU (Device Firmware Upgrade) mode allows low‑level flashing. Attackers use checkm8 to bypass the secure boot chain and load custom firmware.

JTAG, UART, & NAND Mirroring

Hardware debug interfaces are disabled in production devices, but attacks like “checkm8” require no hardware modification — just USB. NAND mirroring (physically reading the storage chip) can bypass passcode limits, but it’s a destructive attack requiring expert soldering and chip‑off tools.

🌐 Apple Ecosystem Attack Surface

iMessage, AirDrop, FaceTime

iMessage is the #1 attack vector for zero‑clicks. AirDrop uses Wi‑Fi and Bluetooth; vulnerabilities have allowed kernel access from nearby devices. FaceTime bugs (e.g., Group FaceTime eavesdropping) demonstrate how complex protocols can fail.

iCloud & Apple ID Phishing

iCloud stores backups, photos, and messages. Attackers phish Apple ID credentials to access this data. Advanced persistent threats combine iCloud access with local exploits for full device compromise.

📖 Legendary iOS Exploit Cases

Pegasus (FORCEDENTRY) – 2021

NSO Group’s zero‑click iMessage exploit. Used a PDF‑rendering bug (CVE‑2021‑30860) to achieve RCE, then a sandbox escape and kernel exploit. Infected devices of journalists, activists, and politicians worldwide. Apple patched all CVEs within months, but the spyware industry continues to evolve.

checkm8 (2019) – Eternal Bootrom Exploit

A vulnerability in the USB DFU stack of A5‑A11 chips. Allows unsigned code execution at the earliest stage. This is unpatchable because it’s in ROM. Used by checkra1n and palera1n jailbreaks. It’s also used by forensic tools to extract data from locked iPhones.

Operation Triangulation (2023)

An iMessage zero‑click chain discovered by Kaspersky. It used multiple exploits, including a kernel vulnerability and a PAC bypass. The attack deployed spyware that exfiltrated microphone recordings, location, and messages. Apple patched with iOS 16.5.1.

Lessons Learned

Zero‑click attacks are the new normal. Defence requires Lockdown Mode, rapid patching, and hardware‑backed attestation. The cat‑and‑mouse game between Apple and exploit developers will never end.

🧠 The iOS Exploit Researcher’s Mindset

“Where does Apple trust the kernel?”

Every exploit starts with a trust boundary: userland → kernel, WebContent → GPU, iMessage → BlastDoor. Find where the system assumes safety, then break that assumption.

“What if I corrupt this pointer?”

PAC makes pointer corruption hard, but not impossible. Think about what data can be forged: file descriptors, Mach port rights, IOSurface references. Every object is a potential primitive.

“How does this crash log reveal ASLR?”

Crash logs contain register state and memory addresses. Use them to defeat Address Space Layout Randomisation. Combine a crash log with a known buffer overflow to build a deterministic exploit.

The Elite Researcher’s Rule

Never stop asking “Why?” — why does this API exist? Why is this entitlement allowed? Why is this buffer on the heap? The answers are where the bugs hide.

🛡️ Advanced iOS Defense & Hardening

Lockdown Mode & BlastDoor

Lockdown Mode restricts attack surface: disables complex WebKit features, blocks most iMessage attachments, and limits wired connections. BlastDoor filters iMessage content in a sandbox. These are the most powerful consumer‑grade protections available.

MDM, App Attestation, & Enterprise Security

Mobile Device Management (MDM) enforces policies. App Attestation validates app integrity before server trust. Use DeviceCheck to detect compromised devices. For enterprises, deploy hardware‑bound MFA and disable sideloading.

Rapid Security Responses & Patch Strategy

Apple’s Rapid Security Response (RSR) delivers patches without full OS updates. Enable automatic updates. Monitor Apple’s security release page for CVE details and update within 24 hours of critical fixes.
