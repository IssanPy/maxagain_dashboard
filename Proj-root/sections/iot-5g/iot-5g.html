🧠 Embedded Systems Internals – The Silicon Foundation

ARM Cortex‑M / A, MIPS, RISC‑V

IoT devices run on microcontrollers (Cortex‑M0/3/4) or application processors (Cortex‑A series). RISC‑V is emerging as an open alternative. Understanding the CPU architecture (registers, memory map, exception vectors) is essential for firmware reversing and exploitation.

# ARM Cortex‑M vector table
0x0000: initial SP value
0x0004: Reset handler
0x0008: NMI handler
0x000C: HardFault handler

Bootloaders, RTOS, and Memory Layout

Devices often use U‑Boot, barebox, or custom bootloaders. Real‑Time Operating Systems (FreeRTOS, Zephyr, ThreadX) handle tasks and interrupts. Memory‑mapped I/O (MMIO) allows peripherals to be controlled via registers. DMA (Direct Memory Access) can be abused to bypass memory protections.

TrustZone & MPU/MMU

Arm TrustZone divides the CPU into Secure and Non‑secure worlds. The Memory Protection Unit (MPU) or Memory Management Unit (MMU) enforces access control. Breaking TrustZone means full compromise of the device's secrets.

💾 Firmware Reversing & Emulation

Extracting Filesystems (squashfs, ubifs, cramfs)

Use binwalk to extract the filesystem from firmware images. Commonly found: hardcoded passwords in /etc/passwd, SSH private keys, MQTT broker credentials, and certificate files.

binwalk -Me router_firmware.bin
cd _router_firmware.bin.extracted/squashfs-root
grep -r "passwd" .

Dynamic Emulation with QEMU

Run extracted binaries on your host using QEMU user‑mode or full‑system emulation. This allows debugging and fuzzing IoT binaries without the physical hardware. Use QEMU with -g to attach GDB.

qemu-arm -L /path/to/sysroot -g 1234 ./vulnerable_binary

Stripped Binaries & Symbol Recovery

Most firmware binaries are stripped of symbols. Use Ghidra's ARM analysis and function identification scripts. Look for references to standard libraries (libc, libssl) to identify imported functions.

🔐 Secure Boot & Trust Chain Failures

Root of Trust & Boot Stages

Secure boot starts with an immutable ROM code that verifies the next stage (usually U‑Boot or a vendor bootloader). Each stage verifies the next using cryptographic signatures. If any verification fails, the device should halt — but implementation bugs allow bypasses.

Rollback & Signature Bypass Attacks

Attackers can downgrade firmware to a vulnerable version if rollback protection is missing. Signature verification may be flawed: accepting self‑signed certificates, ignoring hash mismatches, or verifying only part of the image.

TPM & Measured Boot in IoT

Trusted Platform Modules (TPMs) store hashes of every boot component. Remote attestation allows a server to verify the device's integrity. Compromising the TPM requires physical access or side‑channel attacks.

⚡ Hardware Side‑Channel Attacks

DPA / CPA – Power Analysis

Differential Power Analysis measures power consumption during encryption. Each key bit correlates with a power trace, allowing full key recovery. ChipWhisperer is the open‑source tool for learning DPA.

# ChipWhisperer capture
python capture_trace.py --target aes

Fault Injection – Voltage, EM, Laser

Voltage glitching temporarily drops the supply voltage to skip instructions. Electromagnetic Fault Injection (EM‑FI) induces faults without physical contact. Laser fault injection targets specific transistors. All can bypass password checks or extract firmware.

Decapping & Silicon Probing

Decapping uses acid to expose the silicon die. A microprobe can read memory buses or inject signals. This is the most advanced hardware attack, requiring a forensic lab.

📻 Radio Frequency Engineering for Hackers

Modulation, OFDM, and Signal Propagation

QPSK, 16‑QAM, 64‑QAM encode bits into phase and amplitude. OFDM divides the channel into subcarriers. Understanding these is essential for baseband analysis and signal spoofing. Multipath fading and Doppler shift affect mobile signals.

Antenna Design, Beamforming, and Spectrum Analysis

Directional antennas (Yagi, panel) focus energy for long‑range attacks. Beamforming uses phased arrays to steer signals. A spectrum analyzer (or SDR) visualizes the RF environment, revealing hidden transmitters.

RF Fingerprinting & Jamming

Every transmitter has unique imperfections (clock skew, phase noise) that can be used to identify devices. Jamming can be targeted (specific protocol) or broadband. Both are heavily regulated — test only in Faraday cages.

📱 Baseband Security – The Hidden Processor

Qualcomm, MediaTek, Samsung Shannon

The baseband processor runs its own RTOS and handles all cellular communication. It is the most privileged component on a mobile device. Vulnerabilities allow remote code execution via a malicious SMS or rogue base station.

Baseband Isolation & RRC Parsing

Basebands are supposed to be isolated from the application processor, but shared memory interfaces can be abused. Radio Resource Control (RRC) parsing bugs are the most common entry point — a single malformed packet can compromise the modem.

Tools: MobileInsight, QCSuper, SCAT

Capture and analyse baseband logs with MobileInsight (Qualcomm). Use QCSuper to access the Qualcomm diagnostic interface. These tools are essential for reverse‑engineering baseband protocols.

🌐 Telecom Protocols – The Signaling Backbone

NGAP, GTP‑U, PFCP, SIP, Diameter

5G Next Generation Application Protocol (NGAP) between gNB and AMF. GTP‑U tunnels user traffic between RAN and UPF. PFCP controls packet forwarding rules. SIP handles VoLTE/VoNR calls. Diameter is the authentication and charging protocol.

Vulnerability: Unauthenticated PFCP messages can redirect user traffic or crash the UPF, disrupting all data plane traffic.

SS7 & Diameter Attacks

SS7 (Signaling System No. 7) is the backbone of 2G/3G/4G networks. Attackers can intercept SMS for 2FA bypass, track location, or redirect calls. Diameter has similar flaws in LTE. Both require access to the SS7/GRX network, which is available to rogue carriers.

IMS, VoNR, and Handover Internals

The IP Multimedia Subsystem (IMS) handles voice and video. Handover procedures between 5G, 4G, and Wi‑Fi introduce complex state transitions that can be exploited for DoS or eavesdropping.

🏭 Industrial IoT & SCADA Security

Modbus, DNP3, OPC‑UA, BACnet, IEC 61850

Industrial protocols were designed for reliability, not security. Modbus has no authentication. DNP3 Secure Authentication is optional. OPC‑UA can be hardened, but misconfigurations are rampant. BACnet and IEC 61850 control building and power systems.

# Scan Modbus devices with nmap
nmap -p 502 --script modbus-discover 192.168.1.0/24

PLC Exploitation & OT/IT Convergence

Programmable Logic Controllers (PLCs) can be stopped, reprogrammed, or bricked with a single command. The convergence of OT (Operational Technology) and IT networks creates pathways from the internet to the factory floor. Stuxnet demonstrated the power of this chain.

🏙️ Smart City Attack Surfaces

Traffic Systems, EV Charging, Smart Grids

Smart traffic lights can be manipulated to cause gridlock. EV charging stations communicate via OCPP (Open Charge Point Protocol), which often lacks authentication. Smart grid meters can be spoofed to under‑report consumption.

Water Systems & Public Surveillance

Industrial water treatment plants use SCADA with decades‑old vulnerabilities. Public IP cameras are routinely exploited by Mirai variants and used for botnets or surveillance.

Example: Oldsmar Water Treatment Attack (2021)

An attacker accessed the plant's TeamViewer, increased the sodium hydroxide dosage by 100x, nearly poisoning the city's water supply. The plant used a shared TeamViewer password and had no 2FA.

🧵 Matter & Thread – The Smart Home Protocol

Matter Commissioning & Ecosystem Trust

Matter uses a secure commissioning process based on PKI. However, misconfigurations in the Commissioner role or weak onboarding keys can allow rogue devices to join the network. Apple, Google, and Amazon all act as trust anchors.

Thread Mesh Poisoning & Border Router Attacks

Thread networks rely on a border router for external connectivity. A compromised border router can manipulate the entire mesh. Routing protocol attacks can partition the network or redirect traffic.

🦠 IoT Malware Ecology & Economics

Linux/ARM Malware & Worm Propagation

Most IoT malware targets BusyBox‑based Linux systems. Worms like Mirai scan for open Telnet ports, brute‑force passwords, and download a payload. Persistence is achieved via crontab, /etc/init.d scripts, or modifying the firmware.

Botnet Monetization – Proxies, DDoS, Credential Stuffing

IoT botnets are rented out for DDoS attacks, residential proxy services (e.g., Faceless, 911.re), or credential stuffing. The botnet‑as‑a‑service economy generates billions of dollars annually.

OTA Persistence & Supply‑Chain Infections

Attackers can poison Over‑The‑Air updates to install malware persistently. This requires compromising the update server or signing keys. Once infected, the malware survives reboots and factory resets.

📦 Hardware Supply Chain Security

Counterfeit Chips & Malicious Fabs

Counterfeit chips may contain backdoors or lack security features. Rogue foundries can modify the design to insert hardware Trojans. Verification requires X‑ray imaging and electrical testing — both expensive and rare.

Firmware Signing Failures & Implants

If the OEM's signing key is stolen (as in the case of some router manufacturers), attackers can sign malicious firmware. Firmware implants can survive OS reinstallation by residing in the UEFI or bootloader.

☁️ Cloud IoT – Backend Exploitation

AWS IoT, Azure IoT Hub, Google Cloud IoT

Misconfigured IoT Core policies can allow unauthorized devices to publish or subscribe to MQTT topics. Shadow device APIs can be abused to overwrite desired states, causing physical damage.

MQTT Broker Hijacking & OTA Abuse

Publicly exposed MQTT brokers often allow anonymous access. Attackers can inject commands, steal telemetry, or redirect firmware updates. OTA pipelines without signed firmware allow malicious updates.

mosquitto_sub -h broker.example.com -t "#" -v

🌐 Telecom Threat Intelligence

SS7 Gangs, SIM Swap Rings, and Rogue Carriers

Criminal groups exploit SS7 and Diameter for SMS interception, 2FA bypass, and subscriber location tracking. SIM swap gangs bribe carrier employees to port a victim's number to a new SIM, hijacking their accounts. Rogue carriers provide access to the global telecom network for fraud and espionage.

Nation‑State Telecom Espionage & IMSI Catchers

Stingrays (IMSI catchers) mimic cell towers to intercept calls and track locations. Nations deploy them at scale. Advanced persistent threats target telecom infrastructure for long‑term intelligence collection.

⛓️ Hardware‑to‑Cloud Kill Chains

Chain 1: Firmware → MQTT → OTA Poison → Cloud Pivot

  1. Extract firmware via UART or download from vendor.
  2. Recover MQTT broker credentials from /etc/mosquitto.conf.
  3. Subscribe to all topics on the broker to discover update channels.
  4. Publish a fake OTA manifest with a malicious firmware URL.
  5. All devices update and become part of your botnet.
  6. Extract AWS IoT credentials from the firmware and pivot to the cloud backend.

Chain 2: Fault Injection → Secure Boot Bypass → Firmware Dump → Key Extraction

  1. Voltage glitch during secure boot verification to skip the signature check.
  2. Boot an attacker‑controlled firmware that exports the raw flash contents.
  3. Extract device‑unique keys from the secure storage partition.
  4. Clone the device's identity and impersonate it on the cloud platform.

🧠 The Edge Researcher's Mindset

“Where is trust established in this device?”

Every device has a trust boundary: the boot ROM, the firmware signature, the OTA update server, the MQTT broker. Find where the chain of trust breaks, and you own the device.

“What happens before authentication?”

Many protocols exchange capabilities before any authentication. 5G attach procedure, ZigBee network join, and Thread commissioning all have pre‑auth phases that can be exploited.

“What if two radios talk at the same time?”

Collision, interference, and desense are physical‑layer attacks. A well‑timed jamming signal can break a cryptographic handshake, force a downgrade, or open a window for packet injection.

The Edge Hacker's Golden Rule

Physical access changes everything. Always secure UART, JTAG, and SWD interfaces. Encrypt your firmware. Verify every update. Assume your device will be torn apart in a lab.

🛡️ Advanced Edge Defense & Hardening

Hardware Roots of Trust & Remote Attestation

Use TPMs or Secure Enclaves for secure boot and key storage. Implement remote attestation so that the cloud verifies the device's firmware hash before accepting its data. This detects firmware tampering in real time.

Runtime Integrity & IoT EDR

Monitor device memory and network behavior for anomalies. Use eBPF‑based agents or lightweight EDR solutions on edge devices. Any unexpected process or network connection should trigger an alert.

Signed OTA, SBOM, and Secure Provisioning

All OTA updates must be signed and verified. Maintain a Software Bill of Materials (SBOM) for every device. Secure provisioning injects unique device keys at manufacturing and never stores them in plaintext.

🧪 Edge Security Labs – From Hardware to Cloud

Firmware Emulation & Reversing Lab

Download a router firmware image (e.g., from TP‑Link or D‑Link). Extract with binwalk, emulate binaries with QEMU, and find hardcoded credentials. Set up a Ghidra project to reverse the ARM binaries.

# Download firmware from vendor site, then:
wget http://download.vendor.com/firmware.bin
binwalk -Me firmware.bin
qemu-arm -L ./squashfs-root ./squashfs-root/bin/busybox sh

5G Core & RAN Emulation Lab

Deploy Open5GS and srsRAN on two VMs. Connect a programmable SIM (sysmoUSIM) or use ZeroMQ virtual radio. Capture PFCP and NGAP traffic with Wireshark. Fuzz the SMF/AMF with malformed packets.

# Install Open5GS
sudo apt install open5gs
# Configure and start
sudo systemctl start open5gs-mmed open5gs-sgwcd open5gs-smfd open5gs-upfd

Side‑Channel Attack Lab (ChipWhisperer)

Use ChipWhisperer‑Lite or Nano to perform a DPA attack on AES. Capture power traces, align them, and recover the key byte‑by‑byte. This teaches you how hardware leaks information.

# ChipWhisperer capture and attack
python cw_capture_aes.py
python cw_analyze.py

SDR Replay & Spoofing Lab

Capture a 433 MHz remote control signal with RTL‑SDR, then replay it with HackRF. Use Gqrx to visualise the spectrum. Experiment with GSM/LTE capture using gr‑gsm or srsRAN.

rtl_sdr -f 433920000 -s 250000 capture.iq
hackrf_transfer -t capture.iq -f 433920000 -s 250000

IoT Cloud Backend Attack Lab

Set up a local MQTT broker (mosquitto). Practice anonymous subscription and publication. Then configure AWS IoT Core with a test device, and try to abuse overly permissive policies.

mosquitto_sub -h localhost -t "#" -v

The Lab's Golden Rule

All experiments must be conducted on your own hardware in an isolated environment. Jamming, transmitting on licensed frequencies, and tampering with production devices are illegal without explicit authorization.
