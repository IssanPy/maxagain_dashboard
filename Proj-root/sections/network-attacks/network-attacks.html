📡 Radio Frequency Physics & Engineering

Antennas, Gain, and Propagation

Antenna types: omnidirectional, directional (Yagi, panel), parabolic. Gain measured in dBi/dBd. Polarization (vertical/horizontal) affects signal reception. Fresnel zones impact link quality.

Example: A high‑gain directional antenna can extend attack range from 100m to 1km.

Signal Strength and Noise

RSSI (Received Signal Strength Indicator) in dBm. SNR (Signal‑to‑Noise Ratio) affects throughput. Attenuation from walls, interference from other networks, and multipath fading degrade signals.

# Linux: check signal strength
iwconfig wlan0 | grep Signal

Modulation and Coding Schemes (MCS)

OFDM uses sub‑carriers (52 for 802.11a/g, 114 for 802.11n/ac). QAM (64‑QAM, 256‑QAM) encodes bits per symbol. Beamforming focuses energy toward clients.

Understanding modulation helps in jamming and signal analysis.

RF Hardware & Adapters

Alfa AWUS036ACH, TP‑Link Archer T3U Plus, Panda PAU09 — known for monitor mode + injection. Chipsets: Realtek RTL8812AU/RTL8812BU, MediaTek MT7612U. Always check chipset compatibility before buying.

lsusb
# Identify chipset, install appropriate driver (morrownr/88x2bu, aircrack-ng/rtl8812au)

Spectrum Analysis

Visualize RF spectrum with tools like Gqrx, SDR#, or RTL‑SDR. Identify interference, rogue APs, and hidden channels.

🔬 802.11 Protocol Deep Dive

Frame Types and Subtypes

Management frames: Beacon (0x08), Probe Request (0x04), Probe Response (0x05), Authentication (0x0B), Association (0x00), Deauthentication (0x0C), Action (0x0D). Control frames: RTS, CTS, ACK. Data frames: QoS Data, Null Data.

Each frame has a Frame Control field, duration, addresses, and body. Understanding every field enables frame forging and precise attacks.

# Wireshark filter: wlan.fc.type_subtype == 0x0c  (deauth frames)

MAC Layer & CSMA/CA

Carrier Sense Multiple Access with Collision Avoidance. Network Allocation Vector (NAV) prevents collisions. RTS/CTS handshake for hidden node problem.

Sequence numbers and fragment numbers reassemble fragments. Retransmissions occur on ACK timeout. Understanding this helps in packet injection and DoS attacks.

MIMO and Spatial Streams

Multiple Input Multiple Output uses multiple antennas for parallel data streams. Beamforming improves signal. MU‑MIMO (Multi‑User) allows simultaneous transmission to multiple clients.

Attackers can exploit MIMO for rogue AP capacity or jamming multiple streams.

🔐 Encryption & Authentication Protocols

WEP: The Broken Legacy

RC4 with 24‑bit IV. IV reuse + weak key scheduling = crackable in minutes. Tools: aircrack‑ng, Wifite.

WPA/WPA2: The Workhorse

4‑way handshake: ANonce, SNonce, MIC, PTK derivation using PBKDF2. PMKID can be extracted from RSN IE in association frames. TKIP (deprecated) and CCMP (AES).

# PMKID capture
hcxdumptool -i wlan0mon -o pmkid.pcapng --enable_status=1

WPA3: The Future

Simultaneous Authentication of Equals (SAE/Dragonfly). Anti‑clogging tokens prevent DoS. Forward secrecy. Transition mode (WPA2+WPA3) vulnerable to downgrade.

Protected Management Frames (802.11w)

PMF prevents deauth/disassoc forgery. Not enabled by default on many networks. Check with airodump‑ng --wps output.

🏢 Enterprise Wireless Security

802.1X & RADIUS

Authentication via Extensible Authentication Protocol (EAP) over LAN (EAPoL). The supplicant (client) authenticates to a RADIUS server. Common methods: PEAP, EAP‑TLS, EAP‑TTLS, MSCHAPv2.

Vulnerability: If clients don’t validate the RADIUS server certificate, attackers can set up a rogue RADIUS to harvest credentials.

Rogue RADIUS & Credential Harvesting

Use hostapd-wpe to create a fake enterprise AP. It captures MSCHAPv2 challenge/response, which can be cracked with asleap or hashcat.

hostapd-wpe hostapd-wpe.conf
# then crack captured hashes
asleap -C challenge -R response -W wordlist.txt

EAP‑TLS Hardening

Mutual certificate authentication prevents rogue RADIUS attacks. Ensure clients are configured to verify server certificates and not accept untrusted CAs.

FreeRADIUS Lab

Set up a FreeRADIUS server for testing. Integrate with hostapd for full enterprise WPA2/WPA3 simulation. Essential for learning enterprise attacks.

sudo apt install freeradius freeradius-mysql

🔐 WPA3: Internals & Attacks

SAE Handshake (Dragonfly)

Uses a Password Authenticated Key Exchange (PAKE) resistant to offline dictionary attacks. Commit and Confirm phases. Anti‑clogging mechanism prevents DoS.

Vulnerabilities: timing side‑channels during encoding of the password element. Dragonblood research uncovered downgrade and information leakage flaws.

Transition Mode Attacks

When WPA3 AP supports WPA2 fallback, attackers can force clients to downgrade to WPA2 and capture the handshake. Use airgeddon or manual tools.

# WPA3 downgrade with hostapd-mana
hostapd-mana /etc/hostapd-mana.conf
# Captured WPA2 handshake can be cracked

Side‑Channel & Timing Attacks

Differences in SAE processing time can leak password information. Patched in recent firmware. Always test with dragontool.

💀 Advanced Attack Techniques

Deauthentication Internals

Deauth frames use reason codes (e.g., 7 = Class 3 frame received from nonassociated station). Can be broadcast or targeted. PMF (802.11w) protects against forgery.

# Craft custom deauth with mdk4
mdk4 wlan0mon d -B AA:BB:CC:DD:EE:FF -S FF:FF:FF:FF:FF:FF

KARMA / MANA Attack

Rogue AP responds to all probe requests, tricking clients into connecting. hostapd‑mana implements the attack. Captures handshakes and credentials.

PMKID Attack (No Clients)

Some APs include PMKID in RSN IE of association response or beacon. Use hcxdumptool + hashcat mode 22000.

Fragmentation & Replay

Attack sequence: obtain a keystream fragment, use it to encrypt arbitrary packets. Tools: fragrouter, aireplay‑ng -5.

Client Isolation Bypass

Enterprise networks often isolate clients. ARP poisoning or forcing traffic through attacker’s AP can bypass restrictions.

🕸️ Rogue AP & MITM Infrastructure

hostapd + dnsmasq + iptables

Create a full rogue AP with internet passthrough for transparent MITM. Use bettercap for SSL stripping and credential sniffing.

sudo apt install hostapd dnsmasq

sudo dnsmasq -C /etc/dnsmasq.conf
sudo iptables -t nat -A POSTROUTING -o eth0 -j MASQUERADE

Use nodogsplash or custom Flask app to show a fake login page. When victim enters password, you capture it and grant internet access.

# Customize splash.html to mimic target captive portal

Bettercap automates ARP spoofing and DNS hijacking. Redirect victims to attacker‑controlled servers.

sudo bettercap -iface wlan0 -eval "set arp.spoof.targets 192.168.1.5; arp.spoof on; net.sniff on; set dns.spoof.domains facebook.com; dns.spoof on"

📻 Software Defined Radio & Signal Analysis

SDR Hardware & Tools

sudo apt install gnuradio gqrx-sdr rtl-sdr hackrf

Record and replay RF signals (garage openers, key fobs). Sweep jamming using HackRF to DoS Wi‑Fi bands.

hackrf_transfer -r capture.iq -f 315000000 -s 2000000
hackrf_transfer -t capture.iq -f 315000000 -s 2000000

Custom Frame Crafting

Use Scapy (Python) to craft arbitrary 802.11 frames. Inject them with Lorcon2.

from scapy.all import *
dot11 = Dot11(addr1="ff:ff:ff:ff:ff:ff", addr2="00:11:22:33:44:55", type=0, subtype=12)
frame = RadioTap()/dot11/Raw(load="Hello")
sendp(frame, iface="wlan0mon", loop=1, inter=0.1)

Relay NFC communication between a victim’s card and a terminal over a distance. Tools: ChameleonMini, Proxmark3.

RFID Frequency Basics

📡 IoT Wireless Protocols

Zigbee & Z‑Wave

Zigbee network key extraction via KillerBee. Z‑Wave S2 downgrade attacks. Both used in smart home devices.

LoRa & LoRaWAN

Long‑range, low‑power. Spoofing join requests, bit‑flipping attacks. Use gr‑lora with SDR.

Thread & Matter

IPv6‑based mesh. Matter adds application layer security. Still emerging attack surface.

ESP32 & IoT Firmware

Extract firmware from ESP32 via UART. Analyze with esptool.py and Ghidra. Hardcoded credentials are common.

esptool.py read_flash 0 0x400000 firmware.bin

🚀 Wi‑Fi 6, 6E, and 7 Security

OFDMA & BSS Coloring

Orthogonal Frequency Division Multiple Access increases efficiency. BSS coloring reduces co‑channel interference. New potential side‑channels for traffic analysis.

6 GHz Band & Target Wake Time

Wi‑Fi 6E uses 6 GHz, requiring new adapters. TWT saves power but may be abused for DoS.

Multi‑Link Operation (Wi‑Fi 7)

Simultaneous transmission across multiple bands. Attack surface: MAC layer complexity, roaming attacks.

🔍 Wireless Packet Analysis

Wireshark Filters & Radiotap Headers

Radiotap header contains signal strength, data rate, channel. Use wlan.fc.type_subtype, wlan.addr, eapol, wlan_radio.signal_dbm.

# Wireshark display filter: show all deauth frames
wlan.fc.type_subtype == 0x0c

Beacon & Probe Analysis

Beacon frames leak SSID, channel, supported rates, RSN IE. Probe requests reveal client’s saved networks. Use airodump‑ng or Kismet to log them.

Handshake Verification

Use cowpatty or tshark to confirm a complete 4‑way handshake. Look for M1, M2, M3, M4 EAPOL frames.

tshark -r capture.cap -Y "eapol"

💾 Router Firmware Reversing & Backdoors

Extracting Firmware

Download firmware from vendor site. Use binwalk to extract filesystem, firmwalker to find credentials, and Ghidra to reverse binaries.

binwalk -Me router_firmware.bin
cd _router_firmware.bin.extracted/squashfs-root
grep -r "passwd" .

Hardcoded Credentials & Backdoors

Many routers have hardcoded admin accounts, SSH keys, or debug backdoors. Example: admin:admin, root:12345, or a hidden service on port 5555.

UART & JTAG Access

Identify UART pins with a multimeter. Connect a USB‑TTL adapter to get a root shell. JTAG can read/write flash memory. Use flashrom to dump firmware.

sudo screen /dev/ttyUSB0 115200

⛓️ Attack Chains & Methodology

The Professional Wireless Pentest Workflow

  1. Recon: passive scanning, hidden SSIDs, client enumeration, vendor fingerprinting.
  2. Signal Analysis: RSSI mapping, channel utilization, interference detection.
  3. Attack Vector Selection: based on encryption type, WPS presence, enterprise vs PSK.
  4. Execution: handshake capture, PMKID, Evil Twin, rogue RADIUS, firmware exploitation.
  5. Post‑Exploitation: MITM, credential theft, lateral movement, network pivoting.
  6. Cleanup & Reporting: remove rogue APs, restore settings, deliver findings.

Example Chain: Public Wi‑Fi Takeover

1) Deauth legitimate AP. 2) Start rogue AP with same SSID. 3) DHCP to victim. 4) Captive portal phishing. 5) Capture credentials. 6) MITM to inject malware or steal cookies.

Example Chain: Enterprise Network Access

1) Identify WPA2‑Enterprise with weak EAP (PEAP). 2) Set up rogue RADIUS with hostapd‑wpe. 3) Deauth users. 4) Capture MSCHAPv2 challenge. 5) Crack password. 6) Use credentials for VPN/Email access.

🛡️ Advanced Defense & Hardening

WIDS/WIPS Deployment

Use Kismet as a Wireless Intrusion Detection System. Configure alerts for rogue APs, deauth floods, and channel anomalies. Integrate with SIEM.

Enterprise Hardening

Enforce EAP‑TLS, disable legacy EAP methods, enable PMF, implement NAC, segment guest/corporate VLANs.

RF Monitoring & Physical Security

Regular site surveys to detect unauthorized devices. Use directional antennas for containment. Educate users about Wi‑Fi phishing.

Firmware & Patch Management

Routinely update AP and controller firmware. Remove default credentials. Disable unused services (Telnet, SSH, HTTP).

📖 Real‑World Exploit Case Studies

KRACK (2017)

Key Reinstallation Attack against WPA2. Exploited 4‑way handshake retransmission to reset nonces. Affected almost all Wi‑Fi clients. Patched in updates.

Dragonblood (2019)

Series of WPA3 vulnerabilities: downgrade to WPA2, side‑channel leaks, DoS. Led to improvements in SAE implementation.

Broadpwn (2017)

Remote code execution in Broadcom Wi‑Fi chips via malformed WME element. Impacted millions of Android and iOS devices.

FragAttacks (2021)

Fragmentation and aggregation attacks against 802.11. Flaws in frame reassembly allowed injection of plaintext frames.

Lessons Learned

Protocol design flaws can persist for decades. Always update, use PMF, and prefer WPA3 where possible.

🧠 Wireless Researcher Mindset

Think About Trust & Assumptions

“What if the client auto‑connects without verifying the BSSID?” “What if PMF is not enforced?” “What if the RADIUS server certificate is not validated?” Question every default behavior.

Roaming & Handoff Exploitation

Clients roam between APs. Attackers can force roams to a rogue AP by sending disassociation frames from the real AP.

Signal Strength Manipulation

Adjust TX power to make your rogue AP appear stronger, tricking clients into connecting. Monitor RSSI to understand client behavior.

⚖️ Legal Frameworks & Operational Security

Radio Frequency Laws

Jamming is illegal in most countries. Transmitting on unauthorized frequencies can violate FCC/ETSI regulations. Always use approved channels and power levels.

Authorized Testing & Contracts

Obtain written permission before testing. Define scope, tools, and limitations. Use a Rules of Engagement document.

Operational Security

Use encrypted drives, VPNs, and anonymization (Tor) when conducting research. Log everything. Never store client passwords in plaintext.

Disclosure Ethics

Follow responsible disclosure: report vulnerabilities to vendor first. Provide 90‑day deadline. Publish after patch.
